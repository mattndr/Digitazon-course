HTML

Glossary for HTML.

HTML semantic structure

An example of a semantic structure of an HTML page.

                    
            <!DOCTYPE html>
            <html>
                <head>
                    <meta>
                    ...
                    <meta>
                    <title></title>
                    <link>
                </head>
                <body>
                    <header>
                        <nav>
                            <h1></h1>
                            <ul>
                                <li><a></a></li>
                                ...
                                <li><a></a></li>
                            </ul>
                        </nav>
                    </header>
                    <main>
                        <section>
                            <h2></h2>
                            ....
                        </section>
                        ...
                        <section>
                            <h2></h2>
                            ....
                        </section>
                    </main>
                    <footer></footer>
                </body>
            </html>

<a>

Represent a link to a web resource. It's a pointer to a url address or a specific page section.
Examples: 


    <a href="#element-id"> Page section </a>
    <a href="/login"> Localhost url </a>
    <a href="www.google.com"> Google url </a>

<article>

A semantic tag that represents an independent, reusable component of a page. For example, it can be a forum post or a store article.

<body>

Contains the content of the page that is displayed on screen.

<div>

A generic container that is useful for grouping elements and applying css rules. It's a block-level element: it has a default margin, starts on a new line and always occupies the entire available width.

<footer>

Represent the section of the body that is displayed at the end of the page. It usually contains informations about the company and some navigation links.

<form>

A group of one or more inputs, whose values can be sent to a specific url defined in the action attribute.
When you want to send some user input data to a server, you need to use a form element.
It allows you to specify an http method (get, post, ...) to use when communicating with a server.
Example: 


    <form action="/login" method="POST">
        <input type="text" name="username">
        <input type="password" name="password">
        <input type="submit">
    </form>

<head>

Contains the following informations:

  • meta-informations,
  • page title,
  • (opt) links to css files,
  • (opt) page styles via a set of css rules defined within a <style> tag;

<html>

Root element of an HTML document.

<link>

Allows you to specify a CSS file for the page. There can be zero or more link tags.
Example: 


    <link rel="stylesheet" href="assets/css/styles.css">

<main>

Represent the main content of the body, the core content. The content of this tag should not be repeated in other sections of the document.

<meta>

Used to add a meta-information to the document. It could, for example, be the description of the page that is diplayed by Search Engines, or a feature of the document.
Examples:


    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="A glossary about HTML, CSS and some common tasks.">

<style>

Contains custom css rules to be applied to the document.
Example:


    <style>
        body {
            margin: 0;
            background-color: blue;
        }
    </style>

<title>

Specify the title of the document.
